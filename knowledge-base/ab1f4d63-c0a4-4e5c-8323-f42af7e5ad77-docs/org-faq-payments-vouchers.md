## FAQ — Payments, Fees & Vouchers

### Payments & Fees

**Q: What are the fees for selling tickets?**
A: There are two fees:
- **Transaction Fee**: 7.0% of the ticket price
- **Convenience Fee**: ₱20 per ticket
By default, the organizer absorbs both fees. You can pass them on to the buyer.

**Q: How do I pass fees on to the buyer?**
A: Use the **Pricing Calculator** in the Registration tab. Toggle on **"Pass on Transaction Fee"** and/or **"Pass on Convenience Fee"** to make the buyer pay those fees.

**Q: What is Test Mode?**
A: Test Mode simulates payments — no real money is processed. A yellow banner shows "Payments are simulated. No real transactions will be processed." Use Test Mode while setting up your event.

**Q: How do I go live with real payments?**
A: On the Event Management page, click the **"Ready to go live?"** button (red). This switches from test mode to live payment processing.

**Q: Where can I see my earnings?**
A: Go to the **Wallet** page (top navigation). The Analytics tab also shows Net Revenue with a "Wallet →" link.

### Vouchers

**Q: How do I create a voucher/promo code?**
A: Go to the **Vouchers tab**, click **Create Voucher**, fill in the promo code, choose the usage type, discount type, amount, dates, and applicable tickets.

**Q: What voucher usage types are available?**
A:
- **Single-Use** — Creates unique codes with your prefix (e.g., CODE-ABC12). Each code works once.
- **Multiple-Use** — One code with a set usage limit (e.g., 10 uses).
- **Unlimited** — One code with no usage tracking.
- **Target Price** — Sets a specific final price regardless of original ticket price (e.g., ₱111 for an 11.11 sale).

**Q: Can I limit a voucher to specific ticket types?**
A: Yes. When creating a voucher, use the **Applicable Ticket Types** section to select which tickets the voucher applies to. You can also use "Select All."

**Q: How do I create an invite-only event?**
A: Go to **Edit Event > Settings** and enable **"Require Voucher Code"**. Then create vouchers for your invited guests. They'll need a valid voucher code to register.

**Q: Can I set a minimum order amount for a voucher?**
A: Yes. The **Minimum Order Amount (₱)** field lets you require a minimum cart total before the voucher can be applied.

**Q: How do I make a voucher active immediately?**
A: Leave the **Start Date** field empty. The voucher will be active as soon as it's created.

**Q: Why isn't my voucher working?**
A: Check these things:
1. Is the status set to **Active**?
2. Is the current date within the Start Date and Expiry Date range?
3. Does the voucher apply to the selected ticket type?
4. Has the usage limit been reached (for Multiple-Use vouchers)?
5. Does the order meet the Minimum Order Amount?
