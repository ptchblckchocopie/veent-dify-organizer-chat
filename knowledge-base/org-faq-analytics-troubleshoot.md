## FAQ — Analytics, Printed Tickets & Troubleshooting

### Analytics & Export

**Q: How do I export my registrant list?**
A: Go to Registrants page and click "Export". Choose CSV, Excel, or Google Sheets.

**Q: What's included in the export?**
A: Registrant info, form responses, ticket details, order data, and check-in status.

**Q: How often do analytics update?**
A: Real-time. Refresh the page to see latest data.

### Printed Tickets

**Q: How do I enable printed tickets?**
A: Contact your admin. The feature is enabled per-event.

**Q: What's the printing fee?**
A: Default ₱20 per ticket, configurable by admin.

**Q: Can I void a printed ticket?**
A: Yes, from the Printed Tickets page. The seat/stock is released back.

### Troubleshooting

**Q: Why can't I edit my event?**
A: Check your role. Editors and Managers can edit; Viewers and Scanners cannot.

**Q: Why isn't my voucher working?**
A: Check: Is it active? Is the date valid? Is there stock? Does it apply to the selected ticket?

**Q: Why are tickets showing as sold out?**
A: Check: Total Stock vs Sold + Reserved + Print Allocated. You may have reservations pending.

**Q: Why can't staff members log in?**
A: They need a Veent account with the same email you invited. They should sign up first.

**Q: Payment isn't showing up, what do I do?**
A: Check Orders with "Payment Processing" filter. Some payments take time to confirm via webhook.

### Quick Reference Checklists

**Creating an Event:**
- Create event with basic details
- Add event images (logo, poster, background)
- Create ticket types with pricing
- Set up registration form(s)
- Configure payment settings
- Create vouchers (if needed)
- Set up seat map (if reserved seating)
- Invite staff members
- Customize email template
- Set event to Live

**Day-of-Event:**
- Create scanning areas
- Set up scanner devices
- Approve scanner stations
- Brief staff on check-in process
- Have offline backup ready
- Monitor dashboard for issues

**Post-Event:**
- Export registrant data
- Review analytics
- Process refunds (if any)
- Mark event as Ended
- Request payout/withdrawal
